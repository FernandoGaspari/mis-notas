Sección 2

Títulos y Párrafos

Etiquetas de sección de contenido

<header>
Encabezado de página. (menú de navegación, logo, etc)
<main>
Contenido principal de la página.
<footer>
Pie de página.
<section>
Es un contenedor que agrupa contenido relacionado.
<article>
Representa contenido independiente. (tiene sentido por sí mismo)
<aside>
Representa contenido relacionado pero no forma parte del contenido principal. (ej: "mira mi canal de youtube sobre...")

Elemento de bloque

Los elementos ocupan todo el ancho disponible de la página aunque su contenido no lo haga, los elementos a continuación saltarán a la siguiente linea.

Etiquetas importantes

<adress>
Se utiliza cuando damos información de contacto.
<blockquote>
Se utiliza para marcar las citas a otros autores o documento. (con atributo cite para poner un enlace al documento original)
<pre>
Se usa para que el contenido se vea tal cual como se escribió.
<div>
Se usa como división del documento, generalmente para dar estilo con CSS o para la delegación de eventos en JavaScript.
<hr>
Para decirle al navegador que cambiaremos de tema.

Elemento de linea (<span>)

Los elementos ocupan el ancho del contenido, los elementos a continuación se pondrán al lado.

Etiquetas importantes

<span>
Equivalente a <div>. Se usa para encerrar contenido (poco) y darle estilo con CSS o localizarlos con JavaScript.
<q>
Equivalente a <blockquote> pero para línea.
<code>
Sirve para encerrar código, que queremos representar visualmente, suele ir unido con <pre>.

Etiquetas útiles

<em>
Énfasis.
<strong>
Más énfasis.
<small>
Menos énfasis que el texto normal.
<br>
Salto de linea. (rompe el texto en un punto en específico)
<wbr>
Salto de linea si hiciera falta. (para palabras largas)
<time>
Representa contenido de fecha y hora. (ej: 18:00)
<sup>
Para representar exponentes. (ej: 22)
<sub>
Para representar textos de química. (ej: H2O)

Sección 3

Atributos

Son valores adicionales que configuran y/o ajustan el comportamiento de los elementos.

Atributos globales

class
Se usa para dar estilos con css.
id
Se usa para seleccionar elementos desde JavaScript y para navegar a través de anclas.
title

Permite guardar algún valor en la etiqueta HTML. (muy importante en JavaScript)

Sección 4

Enlaces (Links)

Conecta una página web con otra página web con un recurso interno o externo. Tienen el atributo obligatorio href (donde se especifica la ruta que queremos obtener) y también el atributo target (configura cómo queremos visualizar lo que solicitemos).

Estos atributos se obtienen mediante la etiqueta <a>. (ancla)

Rutas absolutas

Tienen un protocolo (http o https), se usan casi simpre para rutas externas. (<a href="https://google.com> Ir a Google </a>)

Rutas relativas

  • Relativas al punto donde nos encontramos: Funciona si los elementos enlazados están en el mismo nivel (misma carpeta), si los elementos no estan en el mismo nivel se le agrega ../ al principio de la ruta.
  • Relativas a la raíz del proyecto: Es la mejor opción, se debe especificar todas las carpetas desde la raíz bajando niveles hasta llegar a la ruta deseada. (se agrega / al principio de la ruta)

Atributos de los enlaces

target
Para que el enlace se abra en una pestaña nueva.
download
Descarga el recurso solicitado. (el recurso debe estar en mi mismo servidor)

Estos atributos se colocan despues del valor de href.

Se crea una carpeta llamada "assets" para alojar los recursos descargables.

Navegación interna

Se usa la etiqueta <nav>, dentro se crea una lista (cada lista con etiqueta <a>) con el atributo href cuyo valor será el id que anteriormente se le pondá a cada <article>.

Se pone # antes del valor de href.

Navegación externa

Se crea la etiqueta <nav> y en el valor de cada href se pone la ruta seguido de # con el nombre del id.

Listas

Sirven para listar contenido.

Tipos de listas

<ul>
No importa el orden. (atributo type da estilo a los items: disc, square, circle)
<ol>
Si importa el orden. (atributo type da estilo de numeración: 1, A, a, I, i)(atributo start define dónde empieza la secuencia)
<dl>
<dt>
Término a definir.
<dd>
Descripción del término.

Sección 6

Tablas

Etiquetas de las tablas

<table>
Inicio de toda tabla.
<tr>
Construye una fila.
<td>
Construye una celda. (cantidad de esta etiqueta=cantidad de columnas
  • Título de la tabla: Debajo de <table> se usa <caption>.
  • Cabecera de la tabla: Debajo de <caption> se usa <thead> y debajo de <thead> se pone <th>.
  • Pie de la tabla: <tfoot>.

El cuerpo de la página se encierra en <tbody>.

Atributos de las tablas

colspan
Sirve para fusionar columnas.
rowspan
Sirve para fusionar filas.

Sintáxis: <td atributo="nº de celdas"> Nombre </td>

Agrupar columnas

Se usa la etiqueta <colgroup>, dentro de ésta tantas <col> como columnas querramos agrupar. Para agrupar mas de una columna en una sola etiqueta se usa el atributo span en el que el valor es el número de columnas a agrupar. (el primer <col> es la primera columna de la tabla)

Sección 7

Formularios

Etiquetas de formularios

<form>
Engloba nuestro formulario.
<label>
Donde escribimos el nombre del campo a rellenar.
<input>
Crea un campo para que el usuario interactúe.
<button>
Crea un botón que envía el formulario.

Con el atributo for de <label> podemos asociarlo con su respectivo <input>, solo tenemos que ponerle un id a <input> y ese id en for de <label>.

Tipos de <input>

button
El nombre del botón se establece con el atributo value. En aspecto es igual <button> pero no sirve para enviar formularios. (mayormente se usa en JavaScript)
submit
Para enviar formularios. (el nombre se establece igual que en button)
date
Campo para ingresar día, mes y año.
time
Campo para ingresar hora.
mounth
Campo para ingresar mes.
week
Campo para ingresar semana.
search
Barra de búsqueda.
tel
Campo para ingresar teléfono.
email
Campo para ingresar email.
password
Campo para ingresar contraseñas.
url
Campo para ingresar urls,
color
Para especificar un color.
number
Solo se pueden ingresar números.
range
  • step: Define de cuánto en cuánto se mueve.
  • min: Valor mínimo de la barra.
  • max: Valor máximo de la barra.
reset
Resetea el formulario.

<input> tipo seleccionables

radio
Permite seleccionar una única opción de una lista.
checkbox
Permite seleccionar varias opciones de una lista.
  • Todos los <input> deben tener el mismo name para que solo se pueda seleccionar una opción.
  • El atributo checked sirve para que aparezca marcada alguna opción por defecto.
  • Es necesario agregar el atributo value a todos los <input> con la información que queremos que nos llegue a nosotros.

Elemento select básico

Crea una lista donde podemos seleccionar una o varias opciones. Su sintáxis es:

<select name="lenguajes" id="">
<option value="html">HTML</option>
<option value="css">CSS</option> </select>

Elemento select avanzado

Para ordenar por categorías si es que tenemos muchas opciones; se usa la etiqueta <optgroup> con el atributo label para nombrar la categoría.

Datalist

Similar a select, incluye un filtro para facilitar la búsqueda, para esto ponemos el atributo list en un <input> de tipo list. <datalist> necesita un id y encierra todas las <option>del formulario.

Más elementos para los formularios

<fieldset>
Crea un recuadro que agrupa los elementos de un formulario.
<legend>
Le da un nombre al contenido de <fieldset>
<input type="file">
Se usa para cargar archivos para luego enviarlos. (se usa JavaScript)
<meter>
Es como una barra de capacidad. Requiere atributos id, min, max, value, low, high y optimus.
<progress>
Representa el progreso de una tarea, usa atributos id, max y value.
<textarea>
Se utiliza para introducir texto en un formulario, usa atributos name, id, cols y rows; estos dos últimos cambian el tamaño de la caja. (es una caja de comentarios)

Atributos para formularios

placeholder
Da una pista de lo que el usuario tiene que poner en la zona de input. ej:
required
Hace que un campo sea obligatorio. (se puede desactivar desde la consola, se necesitará más seguridad)
readonly
Convierte el campo en solo lectura. ej:
disabled
Desactiva el campo.
min-max
Mínimo y máximo de un campo numérico.
minlenght-maxlenght
Mínimo y máximo de carácteres de un campo de texto.
selected
Equivale a checked.
autofocus
Simula que el usuario hizo click en un campo.

Sección 8

Contenido embebido

Es el contenido que treamos desde afuera. (imágenes, audios, videos,etc)

Imágenes

Imágenes vectoriales
.svg, no pierden su calidad, recomendable siempre se pueda.
Mapa de bits
  • .jpg
  • .png (8 y 24) si se necesitan transparencias.
  • webp (formato que menos pesa)

Para insertar una imágen en el codigo necesitamos la etiqueta <img> con sus atributos src (ruta donde se encuentra la imágen) y alt. (descripción breve por si la imágen falla al cargar)

Device Pixel Ratio

Relación que existe entre los pixeles del dispositivo y los pixeles disponibles para "pintar" contenido. El dpr en pc es de 1, mientras que en celular puede ser entre 2 y 3. Es recomendable no usar la misma imágen en ambos dispositivos ya que el celular no necesita imágenes grandes.

Atributo srcset

Se usa para decilde al navegador qué imágen cargar dependiendo el dpr, para ésto debemos darle un set de rutas separadas por una , (coma). Podemos condicionar al navegador de cargar una imágen dependiendo el dpr, para ésto ponemos al final del enlace 2x o 3x o el valor que querramos ponerle.

Podemos agregar src abajo con un jpg como respaldo. (como para internet explorer que no admite ni srcset ni formato webp)

Etiqueta <picture>

Funciona igual que srcset pero podémos elegir mas opciones.

Desntro de <picture> ponemos <source srcset> con el set de rutas, pero si o si es obligatorio un respaldo con <img src>jpg.

dentro de la etiquieta <source> podemos poner el atributo media con el valor min-width (valor en pixel), esto hará que dependiendo el zoom que le hagamos a la página la imágen cambiará de tamaño.

Etiqueta <audio>

Cuenta tmabién con el atributo src y funciona de la misma manera. Éstos son algundos de sus atributos:

controls
Muestra el reproductor.
autoplay
Para que el audio comience al cargar la página. (los navegadores actuales bloquean este atributo si no esta silenciado)
muted
Mutea el audio.

Etiqueta <video>

También cuenta con src y funciona de la misma manera.

También usa controls, autoplay y muted.

loop
Para que empiece de nuevo cuando termine. (bucle, esta etiqueta tmb sirve para el audio)
poster
Imágen inicial. (miniatura de youtube)

Etiqueta <iframe>

Sirve para insertar una página web en nuestra página web.

Podemos copiar el código iframe de un video de youtube para que aparezca en nuestra página.

Etiqueta <figure>

Sirve para añadir contenido adicinal, como por ejemplo un título a una imágen, video, o lo que sea con la etquieta <figcaption>.

Etiquetas <meta>

  • meta name = "description" content = "acá va la descripción que aparece en el buscador de nuestra página"
  • meta name = "author" content = "acá va el nombre del autor de la página"

Favicon

Es el icono que aparece en una pestaña al lado del nombre de nuestra página web.

Generamos nuestro favicon en www.favicon-generator.org, luego ponemos todos los favicons en assets y copiamos el código que nos da la página dentro de la etiqueta <head>.

Fontawesome

CDN es un servidor que tiene una copia de lo que estamos solicitando; en éste caso usaremos la página www.fontawesome.com

Para usar una librería de iconos, ponemos nuestros datos (el profesor puso fa-rocket en "got a favorite font awesome icon") copiamos el código que nos da la página en <head>, elegimos el icono y y copiamos el codigo donde lo querramos poner.

Accesibilidad y OGP

Atributos de accesibilidad

tabindex="orden 1,2,3,etc"
Se pone en las zonas donde querramos navegar mediante el tabulador,también sirve para seleccionar párrafos para que lo lea un navegador por voz.
aria-label
Sirve para que el navegador pueda leer más detallado cuando una persona invidente pone el mouse por encima del enlace "leer más"
role
para indicarle al navegador por voz sobre qué elemento nos estamos posicionando.
